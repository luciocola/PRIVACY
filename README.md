# QGIS Plugin Privacy Statements

This repository contains privacy statements for various QGIS plugins. The goal is to provide users with clear and transparent information about how their data is handled by different QGIS plugins.

## Purpose

QGIS plugins may collect, process, or transmit user data in various ways. This repository aims to:

- Provide a centralized location for QGIS plugin privacy statements
- Help users make informed decisions about which plugins to use
- Encourage plugin developers to be transparent about data handling practices
- Comply with privacy regulations such as GDPR, CCPA, and others

## Repository Structure

```
PRIVACY/
├── README.md                 # This file
├── PRIVACY_TEMPLATE.md       # Template for creating new privacy statements
├── CONTRIBUTING.md           # Guidelines for contributing
└── plugins/                  # Directory containing privacy statements
    ├── PluginName1.md
    ├── PluginName2.md
    └── ...
```

## Available Privacy Statements

The following QGIS plugins have privacy statements in this repository:

- [OpenRouteService](plugins/OpenRouteService.md) - Routing and accessibility analysis
- [Plugin Reloader](plugins/PluginReloader.md) - Developer tool for reloading plugins
- [QGIS Cloud](plugins/QGISCloud.md) - Cloud publishing and hosting
- [QGIS2Web](plugins/QGIS2Web.md) - Export to web maps
- [QuickMapServices](plugins/QuickMapServices.md) - Basemap services

## For Plugin Users

If you want to know how a specific QGIS plugin handles your data:

1. Check the `plugins/` directory for the plugin's privacy statement
2. If a privacy statement is not available, consider requesting one from the plugin developer
3. Review the privacy statement before using plugins that may collect or transmit data

## For Plugin Developers

If you are a QGIS plugin developer and want to add a privacy statement for your plugin:

1. Use the [PRIVACY_TEMPLATE.md](PRIVACY_TEMPLATE.md) as a starting point
2. Fill in all relevant sections with accurate information about your plugin
3. Submit a pull request with your privacy statement
4. See [CONTRIBUTING.md](CONTRIBUTING.md) for detailed guidelines

## Contributing
