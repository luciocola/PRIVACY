# Privacy Statement for OpenRouteService

## Overview
This privacy statement describes how the OpenRouteService plugin handles user data and privacy. This plugin provides routing and accessibility analysis using the OpenRouteService API.

## Data Collection
The OpenRouteService plugin collects and transmits:
- API key (if provided by user)
- Geographic coordinates for routing queries
- Routing preferences and parameters
- IP address (collected by the OpenRouteService API server)

## Data Usage
The collected data is used for:
- Processing routing and accessibility analysis requests
- API authentication and rate limiting
- Service improvement and analytics by OpenRouteService

## Data Storage
- **Location**: API keys are stored locally in QGIS settings
- **Duration**: Local settings persist until manually deleted
- Query data is transmitted to OpenRouteService servers and subject to their retention policies

## Data Sharing
When using this plugin, data is shared with:
- **OpenRouteService**: All routing queries are sent to their servers
- HeiGIT (Heidelberg Institute for Geoinformation Technology) operates the service

For details on how OpenRouteService handles data, please see:
https://openrouteservice.org/privacy-policy/

## User Rights
Users have the following rights regarding their data:
- Delete locally stored API keys
- Choose whether to use the service
- Request data deletion from OpenRouteService directly
- Review OpenRouteService's privacy policy for additional rights
